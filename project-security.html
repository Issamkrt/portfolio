Over dit project

Voor een school opdracht heb ik een uitgebreide security audit uitgevoerd op een test netwerk infrastructuur. Het doel was om kwetsbaarheden te identificeren, risico's te beoordelen, en concrete aanbevelingen te geven voor security verbeteringen.

De audit omvatte network scanning, vulnerability assessment, password auditing, social engineering tests, en physical security evaluation. Alle bevindingen zijn gedocumenteerd in een professioneel rapport met executive summary voor management en technische details voor IT teams.

Scope & Methodologie

De audit volgde de OWASP Testing Guide en NIST Cybersecurity Framework methodologie:

  • Information Gathering: Passive en active reconnaissance van de target infrastructuur
  • Vulnerability Scanning: Automated scanning met Nmap, Nessus, en OpenVAS
  • Manual Testing: Handmatige verification en exploitation van gevonden kwetsbaarheden
  • Password Auditing: Strength testing van user accounts met John the Ripper
  • Social Engineering: Phishing awareness testing (met toestemming)
  • Reporting: Comprehensive rapport met risk ratings en remediation steps

Belangrijkste Bevindingen

Critical - Ongepatched Server

Beschrijving: File server draait Windows Server 2016 zonder security updates sinds 2023. Kwetsbaar voor meerdere known exploits including EternalBlue.

Impact: Remote code execution mogelijk, potentieel voor ransomware infection en lateral movement.

Aanbeveling: Onmiddellijk patchen naar laatste security updates. Implement automated patch management.

Critical - Zwakke Wachtwoorden

Beschrijving: 35% van user accounts gebruikt weak passwords (onder 8 characters, geen special characters). Meerdere accounts gebruiken "Password123".

Impact: Brute force attacks succesvol binnen minuten. Unauthorized access tot company resources.

Aanbeveling: Enforce strong password policy (min 12 chars, complexity requirements). Implement multi-factor authentication.

High - Open RDP naar Internet

Beschrijving: Remote Desktop Protocol (port 3389) is direct accessible vanaf internet zonder VPN requirement.

Impact: Constant brute force attacks, verhoogd risico op unauthorized access.

Aanbeveling: Disable direct RDP access. Require VPN connection of implement jump server met MFA.

High - Unencrypted Backup Data

Beschrijving: Network backups worden opgeslagen zonder encryption op shared NAS.

Impact: Als attacker toegang krijgt tot netwerk, zijn alle backups leesbaar. Data breach risk.

Aanbeveling: Encrypt all backup data at rest en in transit. Implement access controls op backup storage.

Medium - Insufficient Logging

Beschrijving: Security event logging is niet gecentraliseerd. Log retention is maar 7 dagen.

Impact: Moeilijk om security incidents te detecteren of forensics te doen na breach.

Aanbeveling: Implement SIEM solution voor centralized logging. Extend retention naar minimum 90 dagen.

Social Engineering Test

Met toestemming van school management heb ik een phishing simulation uitgevoerd:

  • Email verstuurd naar 50 docenten/studenten met fake "password reset" link
  • 18 personen (36%) klikten op de link
  • 7 personen (14%) vulden daadwerkelijk credentials in op fake login page

Conclusie: Awareness training is hard nodig. Users zijn kwetsbaar voor social engineering attacks.

Aanbeveling: Quarterly security awareness training met focus op phishing recognition. Implement email filtering met anti-phishing features.

Recommended Security Controls

  1. Patch Management: Implement automated patching voor alle systems (WSUS/SCCM)
  2. Identity & Access: Deploy MFA voor alle remote access en admin accounts
  3. Network Segmentation: Implement VLANs om server segment te isoleren
  4. Endpoint Protection: Deploy EDR solution op alle workstations en servers
  5. Backup Security: Encrypt backups, test restore procedures quarterly
  6. SIEM Implementation: Centralize logging voor security monitoring
  7. Security Awareness: Quarterly training voor alle users
  8. Incident Response: Develop en test incident response playbooks
  9. Regular Audits: Schedule quarterly vulnerability scans
  10. Physical Security: Implement badge access voor server room

Implementation Roadmap

Week 1-2 (Immediate):

  • Patch alle critical vulnerabilities
  • Enforce strong password policy
  • Disable direct RDP access

Month 1 (Short-term):

  • Deploy MFA solution
  • Implement network segmentation
  • Setup centralized logging
  • Conduct security awareness training

Quarter 1 (Medium-term):

  • Deploy EDR solution
  • Implement SIEM
  • Develop incident response plan
  • Encrypt backup infrastructure

Year 1 (Long-term):

  • Establish security operations center (SOC)
  • Implement continuous monitoring
  • Regular penetration testing (quarterly)
  • Achieve compliance certification (ISO 27001)

Lessons Learned

Dit project heeft me veel geleerd over real-world security challenges. Technische controls zijn belangrijk, maar human factor is vaak de weakest link. Comprehensive security requires: technology, processes, én people awareness.

Het schrijven van het audit rapport was een waardevolle ervaring in security communication - technische details moeten vertaald worden naar business impact voor management.
